# 代码审查与路线图（2026-07-15）

## 结论

`0.3.1` 已达到本地公开源码仓库和四复制人可玩主路径的发布条件。当前版本完成真实游戏加载、中文搜索、皮肤/模型联动、热切换、地面对齐、移动动画和缓存完整性验证。公开仓库排除约 170 MiB 的开发帧、游戏美术、Spine 资源、PRTS 网页构建产物和本地缓存。

本轮并行审查按渲染、动画、资源/文档三个边界进行。高优先级正确性问题已经修复；性能、国际化、细粒度选择和复杂视觉增强保留为后续独立任务。

## 本轮已完成

| 领域 | 问题 | 处理与证据 |
| --- | --- | --- |

| 动画队列 | 状态切换后旧相位可能继续播放 | 新计划从下一段 Begin 开始；压力/死亡计划立即抢占 |
| 共享下载 | 一个复制人的取消令牌会取消其他调用者共用的请求 | 共享任务拥有独立生命周期，每个调用者单独等待和取消；四路并发取消回归通过 |
| 缓存校验 | 索引中有 hash 时只比较长度；同长度损坏可被误认 | 每次命中都复算 SHA-256；索引失配文件不会走“未登记文件收养”路径；同长度损坏回归通过 |

| 公开范围 | 开发目录包含大量游戏资源与网页构建物 | `.gitignore` 排除素材、缓存、生成物和复制的网页 bundle；README 与第三方说明记录边界 |

## 下一阶段优先级

### P1：用户可见质量

1. **正式 i18n**：把设置标题、说明、错误消息和枚举显示从硬编码中文迁移到 ONI `Localization`/`STRINGS` 资源，至少提供简体中文与英文。完成标准是两种游戏语言下 Options、`Ctrl+F8` 和日志提示可验证。
2. **每个复制人独立外观**：全局默认继续存在，在复制人详情面板增加覆盖值。需要明确存档序列化、复制人删除后的清理和全局设置变更语义。
3. **自动模型选择**：基建模型用于日常移动/工作，正面或背面模型用于战斗/特定状态。先建立可复现的 ONI 状态到模型合同，再实现跨模型相位切换。
4. **下载/错误状态 UI**：在设置或复制人详情中显示下载进度、缓存命中、失败原因与重试入口，减少仅依赖 `Player.log` 的诊断成本。

### P1：资源一致性

1. **bundle 原子提交**：atlas、skel 和全部纹理先进入 bundle 暂存区，全部校验后统一登记 Ready，避免 bundle 中途失败留下可见的半套资源。
2. **更早获取租约**：解析 bundle 时先保护已确认资源，降低容量维护与热切换并发时的删除窗口。
3. **永久策略专项验收**：下载后重启、断网命中、策略往返和手动清理都需要保存索引/磁盘证据。
4. **安全重定向规则**：任何放宽或细化 HTTP 重定向白名单的方案都属于网络路径改动，实施前单独确认范围和威胁模型。

### P2：渲染与性能

1. 共享解码后的 atlas、纹理、骨骼数据；每个复制人只保留独立 Skeleton/AnimationState。
2. 为 20 个复制人记录托管内存、纹理内存和帧时间，按数据决定 mesh buffer 与材质缓存优化。
3. 验证 clipping、path constraint、多页、Additive/Multiply/Screen 与非黑双色模型的游戏内视觉一致性。
4. 给帧降级路径增加完整 Begin/Main/End 队列；当前实时 Spine 路径优先。

## 发布边界

- 当前显示名称为 `Arknights Operators（明日方舟干员）`。
- `AmiyaDuplicantMod` DLL、命名空间、目录和 `staticID` 暂时保留，避免破坏已有 Options 与 ONI 模组记录。
- 当前版本按全局选择应用到所有复制人。
- 原创代码当前未授予额外开源许可证；第三方组件分别遵守仓库内许可文本。
- GitHub 源码仓库不包含游戏动画或图片。运行时仍依赖 PRTS 资源服务可用性和用户所在网络环境。
